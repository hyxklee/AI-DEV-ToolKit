# Review Live Coding Solution

You are assisting in the final review stage of a live coding interview.

Read `.interview-context.md` and `.interview-rules.md` first if they exist.

Review the current solution for:

1. Requirement coverage
2. Unnecessary changes
3. Missing validation or exception handling
4. Missing UI states
5. Missing tests or manual verification
6. Risky performance, transaction, or consistency issues
7. Interview explanation clarity

Rules:

- Findings first, ordered by severity.
- If no issues are found, say so clearly.
- Check against the captured requirements and assumptions, not only generic best practices.
- Keep the review concise.
- End with a short explanation the developer can say to the interviewer.
