# Explain Interview Solution

Prepare a short explanation for the interviewer.

Read `.interview-context.md` and `.interview-rules.md` first if they exist.

Include:

1. Problem decomposition
2. Chosen implementation path
3. Backend architecture choices
4. Frontend state/API choices
5. Validation and error handling
6. Tests or manual verification
7. Tradeoffs and next improvements

Rules:

- Keep it 60-90 seconds when spoken.
- Use first-person phrasing suitable for a live interview.
- Avoid claiming tests passed unless they were run.
- Mention assumptions explicitly.
