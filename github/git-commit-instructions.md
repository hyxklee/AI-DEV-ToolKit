## 기본
- 커밋 메시지는 하나의 줄글로 작성합니다
- type: 본문 형태로 작성합니다
- type은 소문자 영어, 본문은 기본적으로 한글을 사용합니다
- 변경된 파일의 내용을 요약하여 작성합니다

## Type 규칙
아래 type을 사용합니다:
feat: 새로운 기능 추가
fix: 버그 수정
docs: 문서 수정
style: 코드 스타일 변경 (포맷팅, 세미콜론 누락 등)
refactor: 리팩토링 (기능 변경 없음)
perf: 성능 개선
test: 테스트 코드 추가/수정
build: 빌드 시스템/의존성 관련 변경
ci: CI 설정 변경
chore: 잡일 (기타 변경, 빌드 제외)
revert: 이전 커밋 되돌리기

## 예시
feat: 새로운 기능 추가
refactor: dto validation 수정
fix: 버그 수정
